## 目标
- 在 `src/renderer/src/types/app.ts` 定义统一数据结构，包含 `meta` 与可选的 `html/css/js`（以及可选 `entry`）。
- 更新 `AppStore.tsx`：将内置 `APPS` 与用户添加应用合并为同一数据结构的列表，用于渲染与打开。

## 类型更新
- 在 `types/app.ts` 新增导出：
  - `export interface AppWithContent { meta: AppMeta; entry?: string; html?: string; css?: string; js?: string }`

## 页面更新
- `AppStore.tsx`：
  - 移除本地 `LoadedApp` 类型，使用 `AppWithContent`
  - `userApps: AppWithContent[]`
  - `allApps: AppWithContent[] = [...APPS.map(a => ({ meta: a })), ...userApps]`
  - 列表渲染：`AppItem` 传 `item.meta`
  - 打开逻辑：新增 `openItem(item: AppWithContent)`，根据是否有 `html/css/js` 决定使用内存内容或 `entry`（默认 `index.html`）

## 兼容
- 保持现有上传逻辑不变（返回 `meta + html/css/js`），仅类型替换为 `AppWithContent`
- 内置应用默认 `entry = 'index.html'`；如后续为内置配置添加 `entry` 可直接生效

## 验证
- 类型检查与 Lint
- 功能：列表统一渲染，打开上传应用走内存，打开目录应用走 `entry` 路径解析
